DiamondPlanner Documentation
============================

Overview
--------
The objective of the Diamond Planner is to simplify the team's current Voronoi planner.
It places virtual nodes around each obstacle that work as waypoints for a robot to go around it
while respecting a given clearance.
The algorithm used for finding the best path through the available nodes from start to goal is A*.

Dependencies
------------
- numpy – numerical computations
- matplotlib – plotting and visualization of the field, obstacles, and paths
- Obstacle class – represents circular obstacles and includes collision checks
- sys.path modification is used to ensure the `src` folder is accessible

---

Global Constants
----------------
CLEARANCE – Minimum distance to maintain from obstacles – 100 mm
ROBOT_RADIUS – Robot radius for collision – 90 mm
NODE_COUNT – Number of nodes sampled around each obstacle – 8
NODE_MARGIN – Offset for nodes inside clearance to avoid touching obstacles – -50 mm
FIELD_X, FIELD_Y – Field dimensions – 9000 × 6000 mm
X_MIN, X_MAX – Field boundaries in X – -4500, 4500 mm
Y_MIN, Y_MAX – Field boundaries in Y – -3000, 3000 mm

Classes
-------

DiamondPlanner
--------------
Handles path planning for multiple robots with static and dynamic obstacles.

Constructor
-----------
DiamondPlanner(obstacles, starts, clearance=CLEARANCE)

- Parameters:
  - obstacles – List of Obstacle instances representing opponents and static obstacles
  - starts – List of starting positions for robots (numpy arrays)
  - clearance – Minimum distance to maintain from obstacles (default: CLEARANCE)

- Attributes:
  - self.nodes – List of valid virtual nodes around obstacles
  - self.all_obstacles – Combined list of static obstacles and robot positions treated as obstacles

- Behavior: Automatically generates a graph of nodes around obstacles

---

Key Methods
-----------

build_graph()
-------------
Generates virtual nodes around each obstacle and filters nodes that are inside another obstacle's clearance.
- Uses a circular ring sampling strategy
- Ensures nodes stay within field boundaries
- Avoids overlap with other obstacles and robots

is_path_free(start, goal)
-------------------------
Checks if a straight line between start and goal is free from collisions.
- Skips checking the obstacles located at start or goal
- Returns True if no obstacles intersect the line

plan_path(start, goal)
----------------------
Computes a collision-free path from start to goal.
- Uses A* search over sampled nodes
- Handles the case where the goal is inside an obstacle’s clearance: replaces goal with closest valid visible node
- Returns a list of waypoints forming the path

plot(starts, goals, paths)
---------------------------
Visualizes the field, obstacles, nodes, robot positions, goals, and planned paths.
- Obstacles are drawn in yellow, clearances in semi-transparent blue
- Start points are green circles, goals are red circles
- Planned paths are connected with lines
- Uses matplotlib for visualization

Example Usage
-------------
from TeamControl.voronoi_planner.diamond_planner import DiamondPlanner
from TeamControl.voronoi_planner.obstacle import Obstacle
import numpy as np

# Define obstacles (opponents)
obs_centers = np.array([[-2000, -2000], [1000, -1000]])
obstacles = Obstacle.from_numpy_array(obs_centers, 90, [1, 2], isYellow=True)

# Define starting positions (teammates)
starts = np.array([[0, 0], [1000, 1000]])

# Define goal positions (ball)
goals = np.array([[2000, 2000], [2000, 2000]])

planner = DiamondPlanner(obstacles, starts)

# Compute paths
paths = [planner.plan_path(s, g) for s, g in zip(starts, goals)]

# Visualize
planner.plot(starts, goals, paths)

Visualization
-------------
- Obstacles and robot clearances are drawn as circles
- Start positions (green) and goal positions (red) are clearly marked
- Paths are plotted as lines connecting waypoints
- GREEN: start(s)
- RED: goal(s)
- YELLOW: an obstacle's diameter
- PURPLE: clearance around an obstacle

---

Notes
-----
1. The diamond planner is suited for cluttered environments with multiple moving robots
2. Clearance and node sampling parameters (CLEARANCE, NODE_COUNT, NODE_MARGIN) can be tuned to improve path quality
3. Supports multiple robots planning simultaneously and avoids collisions between teammates
4. Visualization is optional but highly useful for debugging and strategy development
5. The path planning will ignore a robot's own nodes.
6. When the ball (goal) is inside another obstacle's clearance, the path will end at the closest node
that is outside the clearance.
7. The code randomly generates the positions of 12 robots divided into two teams, and one goal.
8. If the clearances of two obstacles intersect, the nodes of an obstacle that would be generated inside
the other obstacle's clearance are discarded.
